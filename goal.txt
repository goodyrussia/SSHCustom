ssh.iptables — Routing Script
This is the most critical script. Model it precisely after box.iptables from boxproxy/box. Implement all infrastructure from that script:
Capability probe system (cache in run/iptables.cap.env, invalidate on reboot via boot_id):

cap_tproxy4, cap_tproxy6
cap_socket_match, cap_socket_transparent
cap_conntrack_match, cap_connmark_match, cap_connmark_target
cap_ipset
cap_ip6_nat

Automatic capability downgrade logic:

If TPROXY kernel modules missing → downgrade to redirect, log warning
If connmark missing → disable performance mode optimization, log warning

Four traffic interception modes:
Mode 1: redirect (default, most compatible, best for SSH)

nat table only, REDIRECT --to-ports $redir_port
TCP intercepted, UDP bypasses (SSH can only carry TCP anyway)
DNS: NAT_DNS_HIJACK chain redirects UDP/53 → daemon for TCP DNS resolution through tunnel
SSH server IP itself: always RETURN (never intercept — would cause routing loop)
Intranet subnets: RETURN
QUIC block rules: when quic=disable, add DROP rules on OUTPUT for UDP 443 and UDP 80

Mode 2: tproxy

mangle table, TPROXY --on-port $tproxy_port
TCP + UDP intercepted
Policy routing: ip rule add fwmark $fwmark table $table, ip route add local default dev lo table $table
Performance optimization when connmark available: DIVERT chain for established connections
SSH server IP: bypass via fwmark exclusion

Mode 3: tun

Uses tun2proxy binary creating TUN device via ip tuntap add
No VpnService — pure root-based TUN creation
Policy routing marks route all traffic into TUN device
--bypass <ssh_server_ip> to prevent routing loop
--dns over-tcp since SSH cannot carry UDP DNS
BOX_TUN_BYPASS_PRE and BOX_TUN_BYPASS_OUT chains for core bypass

Mode 4: tun_udpgw

Same as TUN but with --udpgw-server flag pointing to VPS udpgw server
Enables real UDP tunneling through SSH via UDP-over-TCP encapsulation
Requires udpgw_server field in settings.ini

Chain naming convention (prefix all custom chains with SSHC_):

SSHC_EXTERNAL, SSHC_LOCAL, SSHC_DNS_HIJACK, SSHC_DIVERT, etc.

Runtime snapshot (run/runtime.iptables.env): save active config on start, load on stop for targeted cleanup.
Full cleanup on stop — must handle all four modes:
shstop_all() {
  flush all SSHC_* chains from nat, mangle, filter tables
  remove all SSHC_* chain jumps from PREROUTING, OUTPUT, FORWARD
  delete all SSHC_* chains
  delete policy routing rules (ip rule del, ip -6 rule del)
  flush routing tables (ip route flush table $table)
  if tun mode: ip link set tun0 down; ip tuntap del tun0 mode tun
  if bbr enabled: restore original congestion control
  if tcp buffer tuning: restore original values
  restore IPv6 if was disabled
  remove QUIC block rules
}

Step 5 — Go Daemon (sshcustomd)
Build this entirely fresh. The daemon is the core of the module — it handles SSH connectivity, SOCKS5, transparent proxy, and provides a Unix socket API for the Android app.
SSH Client (internal/ssh/)
SSH Transport Modes:
Mode 1: Direct
go// Plain ssh -D equivalent
// Dial SSH server directly
// Open SOCKS5 dynamic forward
Mode 2: SNI
go// Wrap SSH inside TLS with SNI hostname spoofing
// Use Go's crypto/tls — no external binary needed
// TLS ClientHello carries the SNI hostname (carrier bypass technique)
// After TLS handshake, SSH runs inside the TLS tunnel
// Config: sni_host = the SNI hostname to present
Mode 3: SNI + HTTP Proxy
go// First: CONNECT through HTTP proxy (http_proxy_host:http_proxy_port)
// HTTP CONNECT request: "CONNECT ssh_host:ssh_port HTTP/1.1\r\nHost: ssh_host\r\n\r\n"
// If payload_enabled: inject payload into the CONNECT request headers
// After proxy tunnel established: apply SNI TLS wrap
// After TLS: SSH
Payload Injection:
go// payload is a raw string pasted by user
// Support variables: [host] → ssh_host, [port] → ssh_port, [cr] → \r, [lf] → \n, [crlf] → \r\n
// Inject into HTTP CONNECT headers or as pre-SSH data depending on mode
// Parse and substitute variables before injection
Channel Pool (speed boost):
go// When channel_pool=true:
// Pre-open N SSH channels (default 8) after connection established
// Pool goroutine maintains minimum available channels
// When SOCKS5 request arrives: grab pre-opened channel from pool (no negotiation latency)
// Pool replenishes in background after each channel consumed
// Pool size configurable, default 8
// On connection drop: drain pool, reconnect, rebuild pool
Connection resilience:

Auto-reconnect with exponential backoff (1s, 2s, 4s, 8s, max 30s)
ServerAliveInterval=30, ServerAliveCountMax=3
Reconnect triggers iptables re-apply if rules were lost

SOCKS5 Listener (internal/proxy/socks5.go)

Listen on 127.0.0.1:$socks_port (default 1080)
Full SOCKS5 protocol (RFC 1928)
CONNECT command: open SSH channel from pool → forward TCP
UDP ASSOCIATE: only if proxy_udp=true AND SSH supports it (with OpenSSH it doesn't — return error gracefully)
No authentication required (local loopback only)

Transparent Proxy Listener (internal/proxy/transparent.go)

Redirect mode: listen on 0.0.0.0:$redir_port, read original dst via SO_ORIGINAL_DST getsockopt
TPROXY mode: listen with IP_TRANSPARENT socket option on 0.0.0.0:$tproxy_port
For each intercepted connection: open SSH channel from pool → forward TCP to original destination
DNS interception: when dst port 53 and dns_hijack_tcp=true → resolve over TCP through SSH channel

Critical Requirements Summary
These must be true when done:

ssh.service stop leaves the device in a 100% clean state — no leftover iptables rules, no policy routes, no TUN devices, no fwmarks. Any other VPN app must work immediately after stop.
The Go daemon compiles to a single static binary with zero shared library dependencies (CGO_ENABLED=0), runs on Android 9+ arm64 without any additional runtime.
All four traffic modes (Redirect, TPROXY, TUN, TUN+UDPGW) must work independently. Mode selection in settings.ini determines which is applied. Capability probe auto-downgrades gracefully with clear log warnings.
The Android app never crashes on devices without root — show a clear "Root not found" screen instead.
QUIC defaulting to disable means on first install, UDP 443 and UDP 80 are blocked on OUTPUT, forcing all traffic through TCP-capable SSH tunnel.
The channel pool is on by default and measurably improves throughput on high-latency SSH connections by eliminating per-connection channel negotiation overhead.
The tunnel uptime timer on the Home screen updates every second while connected, showing HH:MM:SS format after the first minute.
Stop completely before applying new settings — never apply partial config changes while tunnel is active.
All custom iptables chains use SSHC_ prefix to avoid collisions with other modules (especially other box-based modules).
The app's Start/Stop/Restart buttons on Home screen are the primary control interface — they call the module's ssh.service script via libsu RootService.
