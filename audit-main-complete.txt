CRITICAL BUGS FOUND:
-------------------
1. SHELL INJECTION (app + module):
   - MainViewModel.fallbackWriteSettings() escapes only \, ", `, $, |.
     Missing: &, ;, <, >, newline, tab, parenthesis.
   - SSHControlService.writeSetting() same incomplete escaping.
   - MainViewModel.startVpnChain() escapes only single quotes in config name.
   Impact: Malicious profile fields or config filenames can execute arbitrary
   shell commands as root.

2. VERSION MISMATCHES:
   - app/build.gradle.kts: versionName "2.3.5" vs VERSION file "2.3.12".
   - module/customize.sh: hardcoded "v2.0.0" vs VERSION "2.3.12".
   - module/update.json: version "v2.0.0", versionCode 20000 vs 231200.
   - daemon/main.go: hardcoded "2.0.0" (overridden by CI ldflags, but local
     builds report wrong version).

3. MISSING FILES (VPN Chain feature broken):
   - module/scripts/ovpn.service — referenced but does not exist.
   - module/scripts/vpnchain.iptables — referenced but does not exist.
   Impact: VPN Chain (OpenVPN over SSH) cannot start or manage iptables.

4. SECURITY — WORLD-ACCESSIBLE SOCKET:
   - daemon/internal/api/unix.go: Unix socket created with 0666 permissions.
   - Any app on device can connect and send control commands (start/stop tunnel).
   - Should use 0660 with shared group, or add authentication.

5. MEMORY/FD LEAK:
   - MainViewModel.pingLatency(): if Socket.connect() throws exception,
     sock.close() is NOT called in catch blocks. Socket leaks.

6. PARSING BUG (daemon config):
   - config/config.go Load(): skips lines containing '(', ')', '{'.
   - If password or payload contains these characters, the setting is silently
     ignored.

7. SYSTEM STATE MUTATION (module):
   - ssh.iptables enable: disables Android captive portal detection.
   - ssh.iptables disable: restores hardcoded defaults, not user's original
     settings. User's custom captive portal config is lost permanently.

8. TUN MODE BROKEN:
   - ssh.iptables: `tun_device` variable is never defined in settings.ini.
   - All `ip link set "${tun_device}"` commands fail with empty device name.

9. DOUBLE-START RISK (boot):
   - service.sh autostart: if curl health check fails, falls back to script start.
   - ssh.service has PID checks, but race window exists during daemon startup.

10. PASSWORD STORAGE (app):
    - Profile.password stored in plaintext SharedPreferences JSON.
    - No encryption or Android Keystore usage.
